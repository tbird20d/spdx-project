Here is some information about my 'sdpx project'

This is a project to analyze SPDX headers in the Linux kernel.
The goals are as follows:
 - add missing SPDX-License_Identifier lines to files
   - prioritize files that are actually included in kernel builds
 - the goal is to be able to build a kernel with no missing SPDX attributes

= tools =
 - ~/bin/scan-for-missing-spdx.sh
   - scan a source tree for files missing an SPDX header
     - allows showing a list of unique SPDX-License-Identifier lines
     - normally, this coalesces similar license identifiers, but this can
       be overridden

 - ~/work/torvalds/create-spdx-lists.sh
   - generate a list of all files and source files missing SPDX files
 - ~/work/torvalds/scan-build-log.sh
   - scan a build log for source files referenced (*.[chS])
 - ~/work/torvalds/check-files-for-spdx.sh
   - read a list of build source files, ad check each one to see if it
     has an SPDX ID line or not
   - output is CSV where the 2nd field indicates the result
 - ~/work/torvalds/gen-spdx-top-dir-report.sh
   - generate the status of the directories one level deep from the
     current directory, using spdxcheck.py

= license addition/conversion policies =
 - add license identify line
   - in format // SPDX-License-Identifier: GPL-2.0-only
   - as top line in C files, as shown with modern line comment syntax

 - conversion rules:
   - GPL mentioned in header without any qualifier becomes "GPL-2.0-only"
   - no license mentioned or referenced in header becomes "GPL-2.0-only"
   - COPYING mentioned or referenced in header becomes "GPL-2.0-only"

 - leave copyright lines
 - Remove lines attributing license, e.g. "This file is subject to ..."

 - where there is ambiguity, and a source is referenced, check
   the license of the source (from where this file was derived)
 - check history (logs) in: historical git repositories
   - try to determine where the code came from (sometimes it comes from
     refactoring existing code)
   - depending on the year of contribution, or the name of the contributor
     that the code is submitted as GPL-2.0-or-later (according to the
     COPYING file that the kernel had for a long time)

= extra conversion notes =
 - Q: can we convert LGPL 2.1 to GPL-2.0?
   - TL;DR - yes, but don't do it
   - Yes, technically. See this clause, section 3 in LGPL 2.1:
-------------------------------
3. You may opt to apply the terms of the ordinary GNU General Public License
instead of this License to a given copy of the Library. To do this, you must
alter all the notices that refer to this License, so that they refer to the
ordinary GNU General Public License, version 2, instead of to this License. (If
a newer version than version 2 of the ordinary GNU General Public License has
appeared, then you can specify that version instead if you wish.) Do not make
any other change in these notices.

Once this change is made in a given copy, it is irreversible for that copy, so
the ordinary GNU General Public License applies to all subsequent copies and
derivative works made from that copy.

This option is useful when you wish to copy part of the code of the Library
into a program that is not a library.

source: https://www.gnu.org/licenses/old-licenses/lgpl-2.1.en.html
---------------------------------------------------------------------
  - However, there are files in the kernel tree already that are LGPL 2.1
    - It may be considered rude by the original author to re-license the file
    - By statically linking an LGPL 2.1 file into a GPL 2.0 program, the
      legal effect is identical to re-licensing the LGPL 2.1 file as GPL 2.0

-----
 - Q: what can be done with files that have extra restrictions?
   - e.g. must mention these authors (or something like that)?
 - Q: what case should AND, OR or WITH be?
   - upper case, see appendix D in https://spdx.github.io/spdx-spec/v2.2.2/SPDX-license-expressions/
 - Q: how should exceptions be expressed?
   - using a WITH <exception> clause in the license
   - see https://spdx.github.io/spdx-spec/v2.2.2/SPDX-license-list/#a2-exceptions-list for a list of standard exceptions
